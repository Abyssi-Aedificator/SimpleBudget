Local Network Sync

Sync your data between devices on the same Wi-Fi network. No cloud, no accounts — data goes straight from device to device.

Sync setup

Sync key
A shared secret. Generate it on one device, then enter the same key on your other devices.
Peer address
The IP address of the computer running the sync agent (e.g. 192.168.1.10). Leave blank on the computer itself.
Port
Port the sync agent listens on. Default is 8777.
Agent status
Checking…

How it works

  1. On your computer, run the sync agent: python simplebudget-sync.py (it prints your computer's IP address).
  2. On this page (computer), click Generate to create a sync key. Leave Peer address blank.
  3. On your phone or tablet, open Simple Budget, go to Settings → Sync, enter the same key and your computer's IP address.
  4. Click Sync now on each device. The device with the most recently changed data wins — its data is copied to the others.

Sync only works while both devices are on the same Wi-Fi network and the agent is running. Your data never leaves your network.

Beta 12 versions
beta 0.9.0 June 23, 2026
New
Credit accounts show available credit — Credit accounts with a credit limit set now display remaining available credit (limit minus what's owed) instead of the raw negative balance, in the account list, the dashboard balance chips, and the Upcoming forecast (totals, per-item running balance, warnings, and chart).
Fix
Upcoming starting balance ignored the rest of the current month — switching the Upcoming window to “Next month” showed today's balance as the starting point, skipping over the remaining transactions and recurring charges still due before month end. The starting (and ending) balance now folds in everything between today and the start of the selected window.
beta 0.8.6 June 23, 2026
Change
Unconfirmed recurring transactions excluded from balances — auto-generated recurring transactions that have not been matched to a real bank import yet (recurring rule only, shown with the plain ↻ badge) no longer count toward account balances or the Total. They are treated as projections until a matching import confirms them, so your account balances reflect money that has actually moved. They remain visible in the transaction list and the Upcoming forecast.
beta 0.8.5 June 17, 2026
New
“Needs review” section on Upcoming — the Upcoming page now shows a Needs review (this month) section listing transactions that already posted this month but still need review. These sit between the future-only forecast and the Transactions list, so nothing pending slips by. The section is informational and does not affect the forecast totals.
Fix
Duplicate upcoming occurrence for nearby linked transactions — a recurring rule still projected its next occurrence even when a transaction had been linked to it on a nearby date (e.g. a rule dated the 19th with a transaction recorded on the 16th). Coverage is now window-based: an occurrence counts as recorded when a linked transaction — reviewed or not — falls within that occurrence’s window, so the duplicate no longer appears in Upcoming or gets auto-posted.
beta 0.8.4 June 16, 2026
New
Local network sync — a new Settings → Sync tab lets you sync your data between devices on the same Wi-Fi network, with no cloud and no accounts. Generate a sync key, run the small simplebudget-sync.py agent on your computer, enter the key and your computer’s address on your phone or tablet, and click Sync now. The device with the most recently changed data wins — its data is copied to the others. Your data never leaves your network.
New
Installable & offline (PWA) — Simple Budget can now be installed as a standalone app and run completely offline. A web app manifest, app icons, and a service worker were added, so once the app is opened from a secure origin (HTTPS or localhost) it caches itself and launches with no network at all — no browser bars, its own icon, on Android, iOS, and desktop.
beta 0.8.3 June 16, 2026
New
Amount search — the transaction search box now matches by amount in addition to description. Typing a value such as 22.91 surfaces every transaction whose formatted amount contains those digits, while plain text still filters by description as before.
New
“Every 4 weeks” recurring frequency — a new Every 4 weeks option is available in the recurring transaction frequency dropdown, sitting between Every 2 weeks and Monthly. Each occurrence advances by exactly 28 days from the start date.
Fix
Amount search broke description filtering — when the search term contained no digits, stripping non-numeric characters produced an empty string that matched every transaction, making description search return all results instead of filtering.
Fix
Rule badge shown on manually renamed transactions — the ❖ badge appeared whenever an imported transaction’s description differed from its original imported text, even when the change was made manually by the user. Renaming is now tracked explicitly: the badge only appears when an import rule made the change, and manual edits clear the flag.
